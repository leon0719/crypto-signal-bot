# 下單推薦 (order-recommendations)

## Overview

當使用者想下單時,用本專案自己的訊號引擎即時掃描主流幣,推薦**幾個可進場的標的與方向(做多/做空)**,附進場參考價、停損、停利。這是技術面輸出,**不是投資建議**。

核心原則:**只推薦引擎給出有效方向(LONG/SHORT)且通過三重確認的標的;沒有訊號就誠實說沒有,絕不硬湊。**

## 流程

1. **一定先重新跑掃描**(行情隨時在變,嚴禁用舊輸出/記憶中的價位):
   ```bash
   bun run scripts/scan-market.ts
   ```
2. 讀每一行輸出:`幣種 → 有效方向 → score → 4h型態 → adx → htf1d → oi → price → atr → 衝突標記`。
3. **篩選可進場標的**:只取「有效方向」為 `SHORT` 或 `LONG` 的(已通過量能+MTF+OI 三重確認)。
   - `NEUTRAL` → 略過(含分數很極端但被量能過濾降級的,代表動能未確認,不可追)。
   - `觀望(降級)` 或帶 `⚠HTF反向`/`⚠OI反向` → 略過(方向衝突)。
4. **排序**:同方向中,4h 型態為「趨勢」且 ADX 高者優先(動能強);大週期 htf1d 同向且絕對值大者加分。挑 3~4 個首選。
5. 為每個推薦標的算進出場位(方向用 price,ATR 同單位):
   - 做空:停損 = price + 2×ATR;停利 ≈ price − 3×ATR。
   - 做多:停損 = price − 2×ATR;停利 ≈ price + 3×ATR。
6. 用下面格式輸出。

## 輸出格式

先一句**免責聲明**,再給表格,再給「不建議進的」與「提醒」。

| 排序 | 幣種 | 方向 | 進場參考 | 停損(2×ATR) | 目標(3×ATR) | 為何入選 |
|------|------|------|----------|-------------|-------------|----------|

收尾固定包含:
- **不建議現在進的**:被量能降級(分數極端但 NEUTRAL)、方向衝突、或大週期反向的標的,各一句原因。
- 若**整個市場一面倒**(例如全部 htf1d 偏空),明說「目前只有做空方向有訊號,逆勢做多無支持」。
- **提醒**:非投資建議、務必照表上停損、急殺易反彈做空別追低、進場前可在 LINE 輸入如 `DOGEUSDT 4h` 看完整卡片再確認。

## Common Mistakes

| 錯誤 | 正確做法 |
|------|----------|
| 直接用上一輪/記憶中的掃描結果 | 每次都重新 `bun run scripts/scan-market.ts` |
| 沒有多單訊號時硬推做多 | 誠實說「無做多訊號」,只推有效方向的 |
| 把分數很空但 NEUTRAL 的當訊號推 | 那是量能不足被降級,動能未確認,排除 |
| 推了帶 ⚠衝突/觀望(降級) 的幣 | 方向衝突一律排除 |
| 只給方向不給停損停利 | 一定附 2×ATR 停損與 3×ATR 目標 |
| 漏免責聲明 | 開頭固定聲明這是技術面、非投資建議 |

## 調整掃描範圍

要改幣種清單或週期,編輯 `scripts/scan-market.ts` 頂部的 `SYMBOLS` / `INTERVAL` / `HTF`(預設 4h 主訊號、1d 大週期確認)。引擎邏輯見 `src/signal.ts`、`src/analyze.ts`。
