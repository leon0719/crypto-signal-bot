這份文件描述**目前實際在跑的 4h 策略**。散在 `signal.ts` / `detect.ts` / `paper.ts` /
`risk.ts` 的規則在此彙整成一處。回測與實驗紀錄見 `strategy-backtest.md`。

最後更新:2026-07-23。

## 1. 執行時機

`scripts/scheduler.ts` 常駐(Docker `detector` 服務),每小時整點後 2 分(UTC)醒來。

- **4h 策略**:UTC 0/4/8/12/16/20 執行,即每根 4h K 棒收盤後兩分鐘。
- **1h 策略**:每小時執行,但 `pushSignals: false`、`liveTrading: false`——純對照組。
  回測顯示 1h 含成本必虧(淨 avgR −0.26、0/8 標的賺錢),**刻意不接任何實際動作**。

策略清單在 `src/strategies.ts`,兩者唯一變因是週期。

## 3. 方向判定

### 3.1 七個評分組件

每個組件輸出 −1 ~ +1:

| 組件 | 權重 | 公式 | 正值代表 |
|---|---|---|---|
| 趨勢 | 2.0 | `(sign(EMA50 − EMA200) + sign(收盤 − EMA200)) / 2` | 價格與中期均線都在長期均線上方 |
| EMA 快慢線 | 1.5 | `sign(EMA12 − EMA26)` | 快線在上 |
| MACD | 1.5 | `sign(MACD 柱)` | 動能偏多 |
| OBV 量能 | 1.0 | `sign(OBV_EMA10 − OBV_EMA30)` | 量能流入 |
| RSI | 1.0 | `clamp((50 − RSI) / 20, −1, 1)` | RSI **低**(超賣) |
| Stochastic | 1.0 | `clamp((50 − %K) / 30, −1, 1)` | %K **低** |
| Bollinger | 1.0 | `clamp((0.5 − %B) × 2, −1, 1)` | 貼近**下**軌 |
| K 棒影線 | 0.5 | 長下影→正、長上影→負(`ta.shadowScore`) | 下影承接 |

前四個是**順勢族**(漲→正分),後四個是**均值回歸族**(跌深→正分)。
兩族方向相反是刻意設計,由 ADX 決定誰說話。

### 3.2 ADX 市況切換

| ADX | 市況 | 順勢族權重 | 均值回歸族權重 |
|---|---|---|---|
| ≥ 25 | 趨勢 | ×1.0 | **×0.15** |
| ≤ 20 | 盤整 | ×0.4 | ×1.0 |
| 20 ~ 25 | 中性 | ×1.0 | ×1.0 |

強趨勢時幾乎關掉均值回歸,避免「持續超買 → 持續喊空」拖累評分。

### 3.3 均線斜率降權

順勢族的淨方向若與 **EMA200 過去 5 根的斜率**相反,順勢族權重再 **×0.5**。
這是打折不是否決。

### 3.4 總分與門檻

```
score = (Σ 組件值 × 權重 / Σ 權重) × 100        範圍 −100 ~ +100

score ≥ +25  → 做多
score ≤ −25  → 做空
其餘          → 觀望
```

## 4. 五道關卡

### 三道否決(直接降為觀望)

| 關卡 | 規則 | 參數 |
|---|---|---|
| 量能 | 當根量 < 20 根均量 → 否決 | `volumeMult: 1.0` |
| 支撐壓力 | 做多但上方 buffer 內有壓力(做空反之)→ 否決 | `srSpan: 5`、`srBufferATR: 0.5` |
| 大週期 MTF | 1d 的 score 與方向**相反** → 否決 | `HTF_MAP` 4h→1d |

### 一道確認

| OI 未平倉量 | OI 的 EMA10 > EMA30(擴張)時,以價格 3 根動能定方向;**只有明確反向才否決**。OI 收縮或抓不到 → 不擋(fail-soft) |

MTF 與 OI 都是**非對抗過濾**:不要求它們同意,只要求它們不反對。

## 5. 進場、停損、出場

訊號確定後以**掃描當下的即時價**為進場價(距 K 棒收盤約 2 分鐘)。

```
做多:停損 = 進場 − 2×ATR    參考目標 = 進場 + 3×ATR
做空:停損 = 進場 + 2×ATR    參考目標 = 進場 − 3×ATR
```

倍數來源是 `signal.ts` 的 `defaultConfig().stopATR / takeATR`——**只有這一個真實來源**,
`detect.ts`、`paper.ts`、`live.ts`、`slack.ts`、`format.ts` 全部讀它。
(2026-07-23 之前這個常數散落四處且與回測不同步,見 `strategy-backtest.md` 事故紀錄。)

### 實際出場是移動停損,不是那個固定目標

紙上交易跑 **`exit: "trailing"`**:

- 停損隨波段極值移動:多單 = 最高點 − 2×ATR,**只上移不下移**
- **沒有固定停利**——讓贏單跟著趨勢跑
- 移動距離 = `PaperConfig.trailATR × 進場當根 ATR`,與初始停損倍數**解耦**

所以卡片顯示的「賺賠比 1.5 : 1」是**參考目標**,真實賺賠分布由 trailing 決定,
單筆可能 +5R 也可能 −1R。

> ⚠️ **實盤路徑目前不支援移動停損。** `live.ts` 只在下單時掛一次交易所端 TP/SL,
> 沒有隨波段更新的機制。因此**紙上跑 trailing、實盤跑固定 TP/SL,是兩套不同策略**。
> 開實盤前必須先實作 trailing(OKX 有移動止盈止損單型),否則驗證的與執行的不一致。

## 6. 部位大小與槓桿

**每筆固定風險 = 開倉時權益的 1%**(`riskPct: 0.01`)。起始 2000 USDT → 每筆最多虧 20 USDT。

```
停損距離 = |進場 − 停損| = stopATR × ATR
名目部位 = 風險額 ÷ (停損距離 ÷ 進場價)
數量     = 名目 ÷ 進場價
```

槓桿由 **ATR 佔價格的百分比**決定(`src/risk.ts`),**不由停損距離反推**:

| ATR / 價格 | 槓桿 |
|---|---|
| ≤ 1% | 5x |
| ≤ 1.5% | 4x |
| ≤ 2% | 3x |
| ≤ 3% | 2x |
| > 3% | 1x |

波動大的幣壓低槓桿,讓強平價離停損夠遠。**槓桿不影響單筆虧損金額**
(那由 1% 風險固定),只影響保證金占用與強平價位置。

### 範例

BTC 65,000、ATR 1,000(1.54%)、做空、權益 2,000:

```
停損     = 65,000 + 2,000 = 67,000      (距離 3.08%)
參考目標 = 65,000 − 3,000 = 62,000
風險額   = 20 USDT
名目     = 20 ÷ 0.0308 = 649 USDT
數量     = 649 ÷ 65,000 = 0.00999 BTC
槓桿     = ATR 1.54% → 3x
保證金   = 649 ÷ 3 = 216 USDT
強平價   ≈ 65,000 × (1 + 1/3) = 86,667   ← 遠在停損之外
```

## 7. 相關性護欄

2026-07 前向測試出現「整批空單一起停損」的出血模式後加入(`detect.ts guardOpportunities`):

| 規則 | 內容 |
|---|---|
| 同輪整批降級 | 同一輪出現 ≥3 支同方向 → 視為同一注,只留 \|score\| 最強一支 |
| 同方向持倉上限 | 同方向最多同時持 3 筆 |
| 實盤額外限制 | 自動倉位總數上限 4、同幣不重複、同一根 K 棒不重複下單(冪等鍵) |
